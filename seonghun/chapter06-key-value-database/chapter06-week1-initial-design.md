키값 저장소 설계

요구사항 정리
- 키 값 쌍 10키로바이트 이하
- 큰 데이터 저장 가능
- 높은 가용성 (장애 발생하더라도 빠른 응답)
- 높은 규모 확장성
- 데이터 일관성 수준 조절 가능
- 정말 낮은 지연시간

요구사항 해석
- 메모리가 아닌 디스크 저장 ( 1. 데이터베이스 명시 2. 큰 데이터 저장 요구 ) 
- 데이터 자동 분할 기능 ( 키/값 쌍 10키로 이하이지만 큰 데이터 저장 가능 요구)
- 파티션 다수
- 파티션 내부 서버 다수
- 내부 자체 캐시 필요
- 디스크 물리적 쓰기 전 완료 

결정사항 정리
1. 비휘발성 저장 장치 저장
2. 10KB 이상 데이터 자동 분할 필요
3. 파티션 최소 3개
4. 데이터베이스 시스템의 진입점은 1곳
5. 매 순간 일관성 챙기지 않고 배치 잡 등으로 추후에 일관성 잡는 구조
6. 설정값 상 최소 쓰기 완료되어야하는 파티션 개수들의 페이지 캐시 저장 후 성공 응답 전송
7. 파티션당 캐시 서버 1개, 데이터 crud서버 1개 이상 


설계

설정값
서버 최소 대수 (최소 3)
서버 최대 대수
서버 활성 대수 (최소 : 서버 최소 대수 )
쓰기 성공 서버 대수 (최소 2 , 최대 : 서버 활성 대수)
